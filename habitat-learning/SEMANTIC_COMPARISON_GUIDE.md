# Habitat-Sim 语义场景对比工具使用指南

## 📖 工具说明

`semantic_comparison.py` 是一个专门用于对比普通 GLB 和语义 GLB 差异的工具。

### 核心功能

1. **并排对比** - 同时显示普通场景和语义场景，方便直观比较
2. **语义标签加载** - 自动加载 `.semantic.txt` 文件，显示详细的物体类别标注
3. **多传感器支持** - 支持切换 RGB、深度、语义三种传感器视图
4. **状态同步** - 两个场景的视角和位置保持同步

## 🚀 使用方法

### 1. 运行脚本

```bash
cd /home/kason/workspace/Tester
python habitat-learning/semantic_comparison.py
```

### 2. 控制说明

| 按键 | 功能 | 说明 |
|------|------|------|
| **W** | 前进 | 控制agent向前移动 |
| **A** | 左转 | 逆时针旋转15度 |
| **D** | 右转 | 顺时针旋转15度 |
| **S** | 切换传感器 | 在 RGB/深度/语义 之间切换 |
| **M** | 切换显示模式 | 在并排/仅普通/仅语义 之间切换 |
| **R** | 重置环境 | 回到初始位置 |
| **ESC** | 退出程序 | 关闭窗口 |

## 🔍 观察重点

### 普通GLB vs 语义GLB 的差异

1. **语义分割精度**
   - 按 `S` 切换到 **语义传感器**
   - 观察两个场景中语义分割的边界是否一致

2. **物体识别能力**
   - 在语义视图中，不同颜色的区域代表不同的物体类别
   - 普通GLB可能只有基础的几何结构
   - 语义GLB应该有更精细的物体级别标注

3. **场景信息完整性**
   - 信息面板显示加载的语义标签数量
   - 标签越多，说明场景中的物体分类越详细

## 📊 语义标签格式

`.semantic.txt` 文件格式：
```
index, object_id, label_name, region_id
1,0E9B48,"floor",1
2,6FC321,"doormat",1
3,5BE5DF,"doormat",1
```

- **index**: 序号
- **object_id**: 物体唯一标识符
- **label_name**: 物体类别名称（如 "floor", "wall", "door"）
- **region_id**: 区域/房间编号

## 🎨 传感器视图说明

### RGB 视图
- 显示真实的彩色图像
- 两个场景看起来应该基本相同

### 深度视图
- 显示距离信息（越近越亮）
- 两个场景的深度信息应该一致

### 语义视图
- **重点观察**：显示物体类别分割
- 普通GLB：可能只有基础的语义信息（墙壁、地板）
- 语义GLB：应该有更详细的分类（家具、门窗等）

## 📈 实际应用场景

1. **数据集质量检查**
   - 验证语义标注的准确性
   - 发现标注错误或缺失

2. **算法测试**
   - 测试语义分割算法在不同场景下的表现
   - 对比不同数据集的语义丰富度

3. **场景理解**
   - 快速浏览场景中的物体分布
   - 了解场景的语义结构

## 🐛 常见问题

**Q: 为什么两个场景看起来一样？**
A: 请确保按 `S` 切换到语义视图，RGB视图下两者确实应该看起来相同。

**Q: 语义视图显示全黑？**
A: 可能是场景文件不包含语义信息，检查是否正确加载了 `.semantic.glb` 文件。

**Q: 程序运行缓慢？**
A: 两个仿真器同时运行会增加GPU负担，可以尝试降低传感器分辨率。
